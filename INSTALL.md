# Installing Ralph Wiggum (Manual)

## The Easy Way (Recommended)

Just tell your AI assistant:

> "Set up Ralph Wiggum in this project using https://github.com/tolulawson/ralph-wiggum"

The AI will read [INSTALLATION.md](INSTALLATION.md) and guide you through an interactive setup.
That setup now installs the full harness: the unified loop, helper libraries,
prompt templates, and vendored planning assets.

---

## Manual Installation

If you prefer to install manually:

### 1. Create Directories

```bash
mkdir -p .specify/memory
mkdir -p specs
mkdir -p scripts
mkdir -p scripts/lib
mkdir -p templates
mkdir -p vendor
mkdir -p logs
mkdir -p history
mkdir -p completion_log
mkdir -p .cursor/commands
mkdir -p .claude/commands
```

### 2. Install The Full Harness

```bash
# Replace this URL if you want to install from a different Ralph fork.
RALPH_REPO_URL="https://github.com/tolulawson/ralph-wiggum.git"
RALPH_BOOTSTRAP_DIR="$(mktemp -d)"

git clone --depth 1 "$RALPH_REPO_URL" "$RALPH_BOOTSTRAP_DIR/repo"

mkdir -p scripts/lib templates vendor

cp "$RALPH_BOOTSTRAP_DIR/repo/scripts/ralph-loop.sh" scripts/ralph-loop.sh
cp "$RALPH_BOOTSTRAP_DIR/repo/scripts/lib/"*.sh scripts/lib/

cp "$RALPH_BOOTSTRAP_DIR/repo/templates/PROMPT_build.md" templates/PROMPT_build.md
cp "$RALPH_BOOTSTRAP_DIR/repo/templates/PROMPT_plan.md" templates/PROMPT_plan.md
cp "$RALPH_BOOTSTRAP_DIR/repo/templates/AGENTS.md" templates/AGENTS.md
cp "$RALPH_BOOTSTRAP_DIR/repo/templates/constitution-template.md" templates/constitution-template.md

rm -rf vendor/speckit-agent-skills
cp -R "$RALPH_BOOTSTRAP_DIR/repo/vendor/speckit-agent-skills" vendor/speckit-agent-skills

chmod +x scripts/ralph-loop.sh

rm -rf "$RALPH_BOOTSTRAP_DIR"
```

This installs the current harness layout:
- `scripts/ralph-loop.sh` as the single loop entrypoint
- `scripts/lib/*.sh` for shared runtime, prompt, provider, preflight, work-item,
  and release-workflow logic
- `templates/` for prompt and constitution templates
- `vendor/speckit-agent-skills/` for deterministic planning assets

Important: client projects should keep a single local copy of the planning assets
under `vendor/speckit-agent-skills/`. Do not also copy this repo's top-level
`skills/` directory into the client project; that directory is only for packaging
and publishing Ralph itself as an installable agent skill.

Recommended: ignore ephemeral runtime artifacts so local logs do not distract from
real code changes:

```bash
touch .gitignore
grep -qxF 'logs/' .gitignore || printf '\nlogs/\n' >> .gitignore
grep -qxF '.circuit_breaker_state' .gitignore || printf '.circuit_breaker_state\n' >> .gitignore
```

### 3. Create Constitution

> **Note:** PROMPT files are auto-generated by the loop scripts. No need to create them manually.

Build mode now treats the constitution as required. The easiest path is to start
from the copied template and tailor it:

```bash
cp templates/constitution-template.md .specify/memory/constitution.md
```

At minimum, make sure your final `.specify/memory/constitution.md` includes:
- Context detection for interactive mode vs Ralph loop mode
- Core principles
- Technical stack
- `## Testing Policy` for exact unit / integration / E2E / device expectations
- Autonomy settings
- Current Ralph loop usage (`./scripts/ralph-loop.sh`, `--runtime`, optional `--model`)
- Completion signal rules covering:
  - `<promise>DONE</promise>`
  - `<promise>ALL_DONE</promise>`
  - `<promise>BLOCKED:reason</promise>`
  - `<promise>DECIDE:question</promise>`
- The work-item release rule: after a review-ready local commit, the runtime owns
  push, draft PR creation, and automatic merge attempts; `awaiting_merge` is only
  the fallback when auto-merge is blocked

Use [templates/constitution-template.md](templates/constitution-template.md) as the
source of truth for the modern contract.

### 5. Create AGENTS.md

```markdown
# Agent Instructions

**Read the constitution:** `.specify/memory/constitution.md`

That file is your single source of truth for this project.
```

### 6. Create CLAUDE.md

Same content as AGENTS.md.

## You're Ready!

1. Create specs with `/speckit.specify [feature description]`
2. Run `./scripts/ralph-loop.sh` to start building
3. Switch runtimes with `--runtime`, for example `./scripts/ralph-loop.sh --runtime codex`

When `work-items.json` is present, Ralph works one task at a time:
it creates a dedicated task branch, pushes that branch only after a successful
implementation pass, opens or updates a draft PR, and attempts to merge it
automatically before starting the next task.

If automatic merge is blocked by permissions or repo rules, the task falls back
to an `awaiting_merge` handoff and Ralph stops until you resolve it.

See the full [README](README.md) for detailed usage.
